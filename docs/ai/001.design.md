# 001. Sandbox SDK 设计

## 目标

提供一个 Go SDK 方法：

```go
CreateSandbox(ctx, opts)
```

创建一个 GitHub Actions runner 作为临时 sandbox，并返回可直接使用的标准 SSH 连接信息。

Phase 1 的目标体验与 `action-upterm` 对齐：

1. sandbox 是 GitHub Actions runner
2. 默认无认证
3. 用户拿到标准 SSH 连接串即可直接连接
4. `Close()` 可结束 sandbox

## 最终方案

Phase 1 直接使用 `upterm host`，不自建 SSH 服务端。

整体流程：

1. SDK 调 GitHub Actions `workflow_dispatch`
2. GitHub 分配一个临时 runner
3. runner 上启动 helper
4. helper 选择 shell：`zsh -> bash -> sh`
5. helper 执行：

```bash
upterm host --accept --server ssh://uptermd.upterm.dev:22 -- <shell>
```

6. helper 执行 `upterm session current`
7. helper 把 SSH 信息写成 metadata
8. SDK 轮询 metadata，返回 sandbox 信息

`Close()` 的实现是取消对应的 workflow run。

## 公开 API

```go
type Sandbox struct {
	ID         string
	Status     string

	Repository string
	Workflow   string
	Ref        string
	RunID      int64
	RunURL     string

	SSHHost    string
	SSHPort    int
	SSHUser    string
	SSHCommand string

	CreatedAt  time.Time
}

type CreateSandboxOptions struct {
	Name string

	GitHubRepository string
	GitHubWorkflow   string
	GitHubRef        string
	GitHubToken      string

	UptermServer string

	StartupTimeout time.Duration
}

func DefaultCreateSandboxOptions() CreateSandboxOptions
func CreateSandbox(ctx context.Context, opts CreateSandboxOptions) (*Sandbox, error)
func (s *Sandbox) Close(ctx context.Context) error
```

约束：

1. 不提供 `Shell` 参数
2. 不提供 `Logger` 参数，日志从 `ctx` 提取
3. 不暴露本地监听相关参数

## 默认值

```go
GitHubWorkflow = "sandbox.yml"
GitHubRef      = "main"
UptermServer   = "ssh://uptermd.upterm.dev:22"
StartupTimeout = 2 * time.Minute
```

`GitHubToken` 优先级：

1. `opts.GitHubToken`
2. `GITHUB_TOKEN`

## Workflow 设计

仓库内提供固定 workflow：

```text
.github/workflows/sandbox.yml
```

Phase 1 固定：

```yaml
runs-on: ubuntu-latest
run-name: sandbox-${{ inputs.request_id }}
```

`workflow_dispatch` 输入：

1. `request_id`
2. `upterm_server`
3. `startup_timeout_seconds`

## Runner Helper

新增 runner 侧 helper：

```text
cmd/sandbox-host/
```

职责：

1. 解析 workflow 输入
2. 按 `zsh -> bash -> sh` 选择 shell
3. 启动 `upterm host --accept`
4. 读取 `upterm session current`
5. 生成 metadata JSON
6. 持续运行直到 workflow 被取消

## Metadata

SDK 通过 workflow artifact 获取结构化结果。

artifact 名称：

```text
sandbox-<request_id>
```

metadata 结构：

```json
{
  "request_id": "xxx",
  "status": "running",
  "ssh_host": "uptermd.upterm.dev",
  "ssh_port": 22,
  "ssh_user": "session-token",
  "ssh_command": "ssh -p 22 session-token@uptermd.upterm.dev"
}
```

workflow 顺序：

1. 启动 helper
2. 等待 metadata 文件出现
3. 上传 artifact
4. 继续等待 helper 退出

这样 SDK 能在 sandbox 仍存活时拿到 SSH 信息。

## SDK 流程

### CreateSandbox

1. 校验参数并生成 `request_id`
2. 调用 workflow dispatch API
3. 通过 `request_id` 或 run id 定位 workflow run
4. 轮询 run 状态，直到 `in_progress`
5. 轮询 artifact，直到出现 `sandbox-<request_id>`
6. 下载并解析 metadata
7. 返回 `Sandbox`

### Close

1. 调 GitHub Actions cancel run API
2. 等待 run 结束
3. 更新 `Status`

## 错误模型

```go
var (
	ErrInvalidOptions       = errors.New("sandbox: invalid options")
	ErrWorkflowDispatch     = errors.New("sandbox: workflow dispatch failed")
	ErrWorkflowStartTimeout = errors.New("sandbox: workflow start timeout")
	ErrMetadataTimeout      = errors.New("sandbox: metadata not published")
	ErrSandboxFailed        = errors.New("sandbox: workflow failed before sandbox was ready")
)
```

## 安全边界

Phase 1 是显式默认无认证。

也就是说：

1. 任何拿到 SSH 连接串的人都可以连接
2. 这与 `upterm host --accept` 的行为一致

当前接受这个产品定义。访问控制不属于 Phase 1。

## 代码规范与技术栈

参考 `https://github.com/117503445/templates`，Phase 1 采用以下约束：

1. 使用较新的 Go 版本，并与模板仓库保持接近
2. 日志使用 `zerolog`
3. 日志对象不通过公开 API 传入，而是从 `ctx` 中提取
4. 目录结构遵循“公开 API 放 `sdk/`，实现细节放 `internal/`，可执行入口放 `cmd/`”
5. 使用 `Taskfile.yml` 组织开发命令
6. 至少提供：
   - `format`
   - `test`
7. 代码整理流程对齐模板仓库：
   - `go mod tidy`
   - `goimports-reviser`
   - `golangci-lint run`
8. 库代码不得直接 `os.Exit`
9. `CreateSandbox` 等公开方法以 `context.Context` 作为日志、取消和超时的统一入口

## Phase 1 交付范围

1. `CreateSandbox()`
2. `Close()`
3. `.github/workflows/sandbox.yml`
4. `cmd/sandbox-host/`
5. 返回标准 SSH 连接串

## 结论

Phase 1 的最终设计就是：

1. sandbox = GitHub Actions runner
2. runner 上直接运行 `upterm host --accept`
3. SDK 通过 workflow artifact 拿回 SSH 信息
4. `Close()` 通过取消 workflow run 回收 sandbox
